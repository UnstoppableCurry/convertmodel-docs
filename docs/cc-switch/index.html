CC-Switch 配置与切换

推荐作为“工具页”使用，而不是主流程入口。它适合桌面端可视化管理多套配置、多供应商和多客户端。

推荐场景

  • 同一台机器上维护多套 API key。
  • 在 Claude Code 与 Codex 间切换工作流。
  • 区分工作环境、实验环境和备用环境。

Claude Code 配置模板

{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "YOUR_API_KEY",
    "ANTHROPIC_BASE_URL": "https://convertmodel.net/anthropic",
    "ANTHROPIC_MODEL": "claude-sonnet-4-6"
  }
}

Codex 配置模板

auth.json
{
  "OPENAI_API_KEY": "YOUR_API_KEY"
}

config.toml
model_provider = "convertmodel"
model = "gpt-5.4"

[model_providers.convertmodel]
name = "convertmodel"
base_url = "https://convertmodel.net"
wire_api = "responses"
requires_openai_auth = true

边界：CC-Switch 更适合桌面系统。无 GUI 的纯服务器环境建议直接管理配置文件。
