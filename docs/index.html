术语与前提

API Key

客户端认证凭据。Claude Code / CodePlan 兼容链路通常使用 Anthropic 分组 key。

Base URL

客户端请求入口地址。示例：https://convertmodel.net。

兼容层

客户端保持原有协议，网关负责模型映射、账号调度与上游路由。

CC-Switch

图形化配置管理工具，适合桌面端维护多 key、多环境、多客户端配置。

三条主线

  1. Claude Code 直连：最短路径，最先完成。
  2. Agents / 并行 / worktree：先理解什么是任务并行，什么是多 agent 并行。
  3. Claude Code vs Codex：先看差异，再决定哪条客户端路径最适合你。
  4. CodePlan 兼容 CC：当你使用 Coding Plan 上游时采用。
  5. Codex 兼容 CC：当你使用 Codex CLI 但希望接入统一网关时采用。

推荐阅读顺序：先完成 Claude Code 直连，再读 Agents / worktree，再看 Claude Code 与 Codex 的差异，最后按需要进入 CodePlan 或 Codex 兼容链路。CC-Switch 作为工具页阅读，不作为主流程前置条件。
